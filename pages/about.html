About WebServ

WebServ is a high-performance HTTP/1.1 server implementation written in C++98, designed to handle multiple concurrent connections efficiently using non-blocking I/O operations.

Project Overview

This project is part of the 42 School curriculum and demonstrates advanced systems programming concepts including socket programming, process management, and HTTP protocol implementation.

Key Characteristics

  • Written in C++98 standard
  • Non-blocking I/O using poll()
  • Support for multiple concurrent clients
  • HTTP/1.1 protocol compliance
  • Configuration file support
  • CGI script execution
  • File upload capabilities
  • Custom error pages
  • Directory listing support

Supported HTTP Methods

  • GET - Retrieve resources
  • POST - Submit data and file uploads
  • DELETE - Remove resources

Architecture

The server uses a modular architecture with the following components:

  • Server - Main server loop and socket management
  • Client - Individual client connection handling
  • ConfigParser - Configuration file parsing
  • Utils - Utility functions for common operations

Configuration

WebServ is configured using a configuration file similar to NGINX. It supports multiple servers, custom ports, root directories, index files, error pages, and location-specific settings.
